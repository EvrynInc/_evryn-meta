## Step 61 — the substantive build (commit `b94fd23`)

**Problem solved:** `processEmail` resolved a forward's gatekeeper purely from the `From` header → a forward from any unregistered address triaged against an empty profile. Now:
- **New table `gatekeeper_inbound_addresses`** (`address` citext PK → `gatekeeper_user_id` FK): many inbound addresses → one canonical gatekeeper record.
- **New `users.outbound_address`** (citext, nullable): the deliberately-set gold-destination, decoupled from inbound; `null` → fall back to `users.email`.
- **`resolveGatekeeperByInboundAddress(address, deps?)`** (db/users.ts): hit → canonical gatekeeper; no-row → null; **DB error OR dangling FK → THROWS** (load-bearing invariant — a swallow-to-null would misroute a real gatekeeper's forward; the throw propagates to `handleNewEmail` → transient → holds the Gmail cursor + retries). Injectable deps for testability.
- **`processEmail` reorder** (process.ts): dedup stays FIRST → `detectForward` moved up → `resolveGatekeeperByInboundAddress` → pure `decideSenderResolution` (`use_gatekeeper` / `escalate` / `find_or_create`).
- **`escalateUnregisteredForward`** (process.ts): a forward from an unregistered lane → a tracked `escalated` `emailmgr_items` row **anchored to the Operator system actor** (`user_id = OPERATOR_USER_ID`; real sender in `metadata.unresolved_from`; `external_id` via `resolveExternalId` so a re-fetch dedups, no re-escalation) + operator `notifySlack`. Short-circuits (no blind triage). The 4h stale-`escalated` checker backstops.
- **`goldDestination(gatekeeper)`** (db/users.ts) = `outbound_address?.trim() || email`. **Lane A owns SET+STASH only — the actual send/threading is Lane B Step 14.**
- **`item.metadata.original_subject`** stashed on registered-lane forwards (process.ts `buildForwardItemMetadata`) — **Lane B reads it.**

**Design calls (all consistent with the balloted design doc `2026.06.17-ac1-step61-...`):** the design was re-verified against the live runtime before building. DC added a **dangling-FK→throw** branch beyond the design (sound — same invariant logic). Operator-anchor for the escalated item confirmed (it's an operator escalation, not a user work item).
