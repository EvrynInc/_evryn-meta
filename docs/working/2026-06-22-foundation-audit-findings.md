## AC0 Consolidated Triage

**Bottom line: the foundation is SOUND.** Two fully-loaded passes (43 meta + 72 deep = 115 findings) found **no hidden rot.** The audit's value is threefold: (1) one marquee *novel* security blocker; (2) ~12 novel non-blockers worth folding in; (3) strong independent corroboration that the existing code is real and the round-2 sprint/lanes are well-aimed — ~15 findings map exactly onto tracked steps, and the two passes repeatedly re-found the same defects (the confidence signal the design wanted). **No "fix-the-foundation-first" mega-pass is needed** — the novel findings route cleanly into the round-2 lanes plus a couple net-new items, and the lanes produce the clean base as they build.

### Tier 1 — Blockers / security (act before/with the lanes)
- **B1. Slack operator-mode has NO sender-identity check** — *[NOVEL, both passes, AC-confirmed against the code].* `slack.ts` routes ANY channel-poster into operator handling (approve sends, read any user's full dossier via a scoped thread, set/rescope scope). ARCH/ADR-014/operator.md all assert a verified-Slack-user-ID gate; the runtime has none (`message.user` captured, never authorized — `config.ts` has no operator-id). v0.2 practical risk is gated by channel hygiene (just Justin in `#evryn-approvals`), but the architectural guarantee is *absent* and the fix is cheap (gate the handler on a config `SLACK_OPERATOR_USER_ID`). → **slack.ts (Lane B's file) — needs an owner/decision.**
- **B2. Tool UUID-validation gap (the un-generalized `record_pass` B-1 fix)** — *[NOVEL, pass-2, high].* `append_pending_note` / `append_cross_user_note` / `set_thread_scope` / `rescope_messages` take an Evryn-supplied `user_id` as a plain string with no UUID/scope validation — the exact fabricated-UUID class `record_pass` was hardened against (B-1), never generalized. A wrong/fabricated UUID lands a write on the wrong user. Cheap (mirror the record_pass fix). → **classify.ts MCP region (Lane B).**
- **B3. M1 silent-death detector is plumbing-only** — *[KNOWN = Step 4 / Lane M1; both passes re-confirm].* Only the env-gated smoke test calls `notifyEmergency`; no condition fires it. Plus a novel sibling: `/health` always returns 200 regardless of liveness (which is exactly what makes the planned external-watchdog path viable). → **Lane M1 (already assigned).**
- *(service_role / `supabase_read|upsert` arbitrary table reach + the prose-only `cross_user_notes`/profile firewall = KNOWN BUILD-S1, correctly v0.3-scoped at single-gatekeeper v0.2; the cheap v0.2 slice of it is B2.)*
