## The keystone (how a halt stays clean + lossless + resumable)
Trip ‚Üí `parkInFlightItemForHalt` parks the in-flight item `processing ‚Üí "new"` (so `ensureItemTransitioned` no-ops; no error-stamp, no per-item `notifyDev`) ‚Üí `runEvrynQuery` returns the sentinel ‚Üí `processForward`/`processDirect` return `"halted"` ‚Üí `planPollBatch` holds the Gmail cursor ‚Üí restart ‚Üí the held cursor re-fetches the email ‚Üí the dedup **resumes** the parked `"new"` row ‚Üí triage re-runs against the existing item (no new row, no duplicate `messages` row). One emergency alert; zero error-storm; zero dropped mail.
