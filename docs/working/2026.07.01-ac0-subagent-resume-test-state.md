## Mechanism note

`SendMessage` reported *"resumed from transcript"* → resume rehydrates the agent's saved transcript (which holds all its `Read` results — that's why the load persists). Two consequences: each resume **re-sends the whole transcript** (cost scales with the agent's loaded size — resuming a heavy agent is not free), and resumability is **tied to that transcript persisting**.
