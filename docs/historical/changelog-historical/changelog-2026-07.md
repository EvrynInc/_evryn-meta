- **The thread/identity model — "one mind, many rooms" (design doc §4.6):** a thread is a room, a wake is the agent stepping into one room fully briefed; there are no floating "versions" because memory (the trunk) + digests/headlines + the ledger ride into every wake. Cross-room learning is a thread-stamped memory note, not a message-to-self. Threads are bounded by TOPIC (email-subject-line discipline); `post_to_thread` moves context between rooms. Digests (rooms you're in) ≠ the little birds (watchers of rooms you're NOT in). — [ACf, Justin's questions settled]
- **SOTA-vetted (design doc §9):** our narrative-memory-file + human-reviewed-consolidation substrate is mainstream-correct and *ahead* on auditability; the field confirms you must gate ambient wakes (LangChain "ambient-agent triage," 40–70% cost cuts) — we do it via membership rather than a classifier.
