# Doc-Update Routing Protocol

> **Truncation check:** the last line of this file should read `FULL FILE LOADED`. If you don't see it, reload or read in sections until you confirm the complete file.
>
> **How to use this file:** This is the **lookup** for keeping Evryn's durable docs aligned with the system as it changes. When you ship or land a change, it answers two questions mechanically: **(1) which surfaces of truth does this change touch?** and **(2) who is allowed to change each of them — can I edit it myself, or must I route it to an owner?** It turns "keep the docs current" from a vague good-intention into a checklist. It is pointed at by the `#lock` doc-currency gate (`docs/protocols/lock-protocol.md`), by the doc-currency headers on `evryn-backend/docs/ARCHITECTURE.md` + the BUILD docs, and by the doc-chain beats in the AC / Soren / Mira manuals.
>
> **What this is NOT:** it is **not** a license to treat the docs as a substitute for reading the runtime. Accurate docs are a trustworthy *on-ramp* — knowing both what we *intend* and what the runtime *actually is*, at a granular level, stays load-bearing. This protocol keeps the on-ramp trustworthy; it never replaces `src/`.
>
> **Do not edit this file without Justin's approval.** Propose changes; don't make them directly.
>
> *Authored 2026-07-14 (AC7, doc-accuracy process lane). Living document — update the registry + matrix as surfaces and routing change (see "Keeping this map current" at the bottom).*

## Why this exists

Evryn's architecture, build, and sprint docs are *supposed* to stay current as we build — but it was hit-or-miss, with no enforcement and no map of *which change goes to which doc section*. So drift accumulated (a live feature stuck on a "blocker" badge; a renamed table the docs never caught up to; a dropped table whose references lingered) until a dedicated reconciliation lane (AC6, 2026-07) had to walk the whole history to fix it.

**A one-off fix doesn't hold. The fix is process-structural and layered** — several independent triggers, because no single reminder reliably fires. This doc is the **anchor** the triggers point at: it makes the update a *lookup*, killing the "I shipped it but didn't know where the doc-home was" failure.

**The goal (hold it precisely):** accurate docs are for **(1) accuracy, (2) shorter / less-duplicated, (3) docs and runtime *agree*** — so an instance stops paying a tax reconciling "what we meant" against "what's real." (Re-read the "What this is NOT" note above — this is an on-ramp, not a runtime-replacement.)

---

## The 30-second workflow

1. **Name the change** in **Part 2 (the change-type matrix)** below — e.g. "a subsystem shipped," "a schema migration," "a new tool."
2. **Hit every surface** that row lists. Don't stop at the obvious one — the whole point is the surface you'd have forgotten.
3. **For each surface, check its owner + gate in Part 1 (the registry):** if it's yours, edit it (honoring its gate — some need Justin's explicit go); if it's someone else's, **route it** (name the section + owner) rather than editing it or skipping it.
4. **Anything you can't update or route right now gets *named* as a routed item** — never left as a silent "I'll get to it." (This is what the `#lock` gate enforces.)

## Part 1 — The Surfaces of Truth (registry)

Every durable surface a system change can invalidate, **grouped by who may change it (from the AC vantage).** A team owner reading this treats *their own* surface as directly-editable and routes the rest — the Owner column is the source of truth; the tiers are the AC-side shortcut.

### Tier 1 — AC updates directly (just do it; no per-edit approval needed)

| Surface | Home of… | Owner | How a change reaches it |
|---|---|---|---|
| `_evryn-meta/CHANGELOG.md` | What was built/changed, cross-repo | AC (standing AC0 priority) | Edit freely — changelog is excluded from the propose-first gate. Changelog-as-you-go. |
| `_evryn-meta/docs/deploy-log.md` | What's **live in prod** (the `v0.2.N` ledger) | AC | Add a newest-first `v0.2.N` row **only on a prod `railway up`** — date + deployed commit + git tag + one-line "what shipped." The **ledger + item STATUS is the authoritative pair** for "did X ship?" — a "riding wave-X" tag is intent, not proof. |
| `evryn-backend/docs/SPRINT-V0.2-HARDENING.md` · `SPRINT-V0.3-CANDIDATES.md` | The live/candidate sprint steps | AC | Mark steps DONE / add steps / reshape. DC/QC don't write here. |
| `evryn-backend/docs/operator-guide.md` | **Justin's operating cheat sheet** | AC (DC flags operator-relevant changes) | Edit directly whenever a change affects how Justin *operates* Evryn (a new env var, a flip like `HAIKU_SCREEN_MODE`, a new go-live step, a new Slack command). A stale operator-guide costs Justin directly. |
| `evryn-backend/docs/schema-reference.md` | The **live DB-schema snapshot** (pulled from Supabase) | AC (or whoever runs the migration) | **Re-pull from Supabase after *any* schema change** — it's a generated snapshot, not authored prose. It drifts silently precisely because nobody owns re-pulling it. |
| `evryn-backend/docs/glossary-phase-*.md` | New technical terms for Justin | AC | Offer Justin new terms when introducing a new concept; add on his yes. |
| `_evryn-meta/docs/decisions/NNN-*.md` (ADRs) | Decisions | Author (anyone) | **Write the ADR at decision time** — excluded from propose-first. A decision captured only in a session doc evaporates under compaction. |
| Linear (EVR workspace) | Discrete task tracking | AC / owner | Create/update tickets per `evryn-team-workspace/shared/protocols/linear-protocol.md`. |

## Part 2 — Change-type → surfaces (the lookup)

Find your change; hit every surface listed; route each per Part 1. The **Gotcha** column is the trap that class of change repeatedly springs.

| Change-type | Surfaces it must reach | Gotcha |
|---|---|---|
| **1. A subsystem / feature ships** (built → merged → deployed) | BUILD phase badge · ARCH status badge + prose · SPRINT step → DONE · deploy-log `v0.2.N` row + git tag *(if a prod deploy)* · CHANGELOG · current-state "Live deploy" line *(if deployed)* · operator-guide *(if operator-facing)* · **identity files → route Mira** *(if it changes a capability Evryn uses)* | **Built ≠ merged ≠ deployed.** Drift went *both* ways — a live feature stuck on "blocker," a shipped-OFF feature wrongly read "live." Use the deploy-log + STATUS pair as the authoritative "did it ship"; don't flip a badge to "live" on a mere merge; a config-gated ship reads "shipped, **inert until `<FLAG>`**," not "live." |
| **2. A schema / DB migration** (add / rename / drop a table or column; new RPC / trigger / RLS) | **schema-reference.md → re-pull** · ARCH Data Model table + prose · BUILD Supabase-Schema quick-ref · ADR *(if the schema decision is architectural)* · pg_dump before+after in `backups/` · the runtime type (`UserRecord` / `ItemStatus` in `src/db/*` — code, out of doc-scope, but the doc must match it) | A **new `public` table needs `GRANT … TO service_role` + `ENABLE ROW LEVEL SECURITY`** *now* (the Oct-30-2026 Data-API cliff). schema-reference.md is the surface everyone forgets — it's generated, so it only stays true if someone re-pulls it. |
| **3. A table / mechanism / function renamed or dropped** | Everything in row 2, **plus: grep every doc AND every `src/` code-comment for the old name** and repoint or remove | This is the `story_history → story_versions` / dropped-`notify_queue` class. **Stale references are silent bugs.** A rename isn't done until the grep is clean. |
| **4. A new / changed runtime capability or tool** Evryn uses | **identity files → route Mira** (`triage.md` / `core.md` / `operator.md` — so Evryn knows about it and routes to it) · ARCH (Agent Architecture / tool list / Identity Composition) · the tool's own description in `src/triage/classify.ts` (code — must agree) · operator-guide *(if operator-facing)* | Identity is **Mira's** — surface the mechanics, don't edit voiced content. An identity file that names a tool/store the runtime won't have *when Evryn needs it* is a real defect (as load-bearing as a code bug); QC's spec-runtime-match check is the backstop. |
| **5. A new / changed env var or operational knob** | operator-guide (how Justin flips it) · current-state Infrastructure / Railway-env line · ARCH or BUILD *(if it gates a subsystem)* · deploy-log *(if it must be set AT deploy or prod boot-fails)* | The **"deploy landmine"** pattern — e.g. `EVRYN_ENV` must be set before the next `railway up` or prod fails to boot (by design). If a knob has a deploy-time precondition, say so at the deploy-log home, not just in prose. |
| **6. An architectural / strategic decision is made** | **ADR written AT decision time** · ARCH references it · BUILD / SPRINT *(if scope changes)* · the specific surfaces the decision touches | Decisions captured only in a session doc get lost under compaction. Write the ADR *before the session ends* — session docs capture the discussion; ADRs capture the decision. |
| **7. Company / strategy truth changes** (Hub-level) | Hub + the spoke → **propose to Justin** · **company-context module (`evryn_knowledge`) refresh → route Justin/Lucas** · ARCH *(if technical direction changes)* | The public-safe `company-context` module is the downstream DB dependency that's invisible because it's a row, not a file — the Hub's own header calls out refreshing it. |
| **8. A doc is moved / renamed** | **grep ALL repos** for references + repoint · leave a **redirect stub** if it's a load-bearing filename | Stale paths are silent bugs. Update every reference **in the same commit** as the move. |

## Keeping this map current (self-maintenance)

This map only works if it stays complete — a routing map that doesn't know about a new surface recreates the exact drift it exists to kill.

- **When you create a new surface of truth** — a new doc, table, or module that becomes a *home* for some class of truth (as AC6 created `BUILD-EVRYN-v0.3.md` + `BUILD-FUTURE.md`) — **add a row to Part 1 and wire it into the relevant Part 2 change-types**, in the same pass. This is itself a Tier-2 edit (propose to Justin).
- **When an owner or gate changes** (e.g. a surface changes hands, or a new agent takes a domain), update the Owner column.
- **`#sweep` reviews it** — the weekly consistency pass (`evryn-team-workspace/shared/protocols/sweep-protocol.md`) should sanity-check that the surfaces here still exist, the owners are current, and no new home slipped in unlisted.
