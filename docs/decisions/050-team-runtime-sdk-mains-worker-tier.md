## Context

The founding team runs manually in Claude Code (ADR-021 paused the original SDK build in favor of Cowork/Claude Code). Autonomy needs a persistent harness. 2026-05-20: an SDK build was designed (feasibility study, same folder as the design doc). 2026-05-30: Justin reversed to model-agnostic on API economics + the no-throttle principle (never throttle agents to save money — make them cheap instead). By 2026-07 the facts moved: (1) Anthropic paused its plan to meter Agent-SDK subscription use — first-party SDK use draws from Max-plan limits today, with capped "extra usage" as the sanctioned paid overflow; (2) the SDK shipped the primitives we'd otherwise hand-roll (hook-based structural permissioning, resumable subagents, structured outputs, rate-limit telemetry, `setup-token` headless auth); (3) Mastra hit 1.0 — right-sized for a worker tier.

## Decision

Two tiers. **Mains (8 founding agents):** Claude Agent SDK on Justin's Max 20x via `claude setup-token`, Opus, one Railway fleet-service, fresh-composed per-thread wakes with logged composition manifests (no long-lived sessions — compaction is not steerable), all comms through an audited `agent_messages` store (Supabase "Evryn Team") with write-time message typing, an **in-house tasks/proposals system replacing Linear**, proposal-protocol authorization enforced by hooks (structurally non-bypassable), per-job hard dollar caps with forecast-based early-warning + Emma's aggregate commitment accounting (Lucas auditing Emma; every kill pings Justin), loop-signature + velocity circuit breakers with warn-before-kill and lossless parking, extra-usage as the only billing overflow — **never model-downshift**. **Workers:** cleared specialists on Anthropic (read-scoped sub-clearances) and contract workers on open models via Mastra/tool-native agents, structurally firewalled (no secrets, no workspace, no DB; sanitized slices only; outputs verified). **Identity is packaged environment-agnostically** (single home in `evryn-team-workspace`; environment adapters + an MCP bridge give interactive Claude Code sessions the same runtime tools — no forked agents; harness code in the new `evryn-team-runtime`). **External contact at any tier is a per-job authorized capability enforced structurally (outbound-contact tools are exposed to a wake only when its job carries the authorization — the same hook mechanism as proposal enforcement), logged; the bright line is on-our-behalf vs. serving-users — end users are served exclusively by the API-billed Evryn product.** Autonomy activates post-Meta-Meeting (EVR-103/104 sequencing).

## Why not Mastra-for-everything (the 5/30 direction)

Subscription billing flows only through Anthropic's own surfaces — Mastra mains would pay ~$300–650/mo now (~$1–2.8K full-tilt) vs. ~$0 marginal on the already-paid Max plan; the SDK is the full Claude Code engine the mains would otherwise re-implement; and the secret-holding tier stays on a first-party harness rather than a large open-source dependency tree. The 5/30 no-throttle principle is preserved by cheap workers + extra-usage overflow, not by cheapening the mains. (Soren, reviewing as the superseded direction's author: "the PRINCIPLE survives intact; only the application re-derived" — and his 5/30 seam-objection, that the Claude SDK is lossy on open-model tool-use, is honored by running open models natively on Mastra where they live.)

## Flip-triggers (recorded so the door stays open)

Anthropic un-pauses metering at rates that erase the mains' edge, restricts headless subscription auth, **or clarifies/enforces its terms against fleet-style subscription use** → the mains' tier re-opens and Mastra-everything is re-evaluated (cheap pivot: the worker tier and the auth seam already exist). **These triggers have a named watcher:** Soren owns the watch-list entry (`evryn-team-workspace/shared/projects/product/watch-list.md`), Emma on the economics half; the `--bare`-CLI watch item folds into it. **S6 is a GATE, not an afterthought: it runs before any confidential material (Nathan's never-leak categories — above all privileged legal material and the identity files) flows through the subscription path; a negative S6 finding re-opens the mains' substrate decision (commercial-API ZDR fallback for confidential work) before Phase 1 proceeds with loaded identities.**

## Consequences

(+) near-zero marginal main-tier cost; proven engine; audit-complete comms; structural authorization; bounded-mistake safety; worker economics ~5–25× under Haiku; one work-tracking system with no per-seat cost.

(−) residual policy gray on fleet-scale personal-plan automation (mitigated: the bright line, internal-default posture, budget governor, full logging); subscription policy risk (flip-triggers — and the plan already prices the likely future: budget as if a ~$200/mo metered credit arrives); **a single-subscription single-point-of-failure** — one account action would halt all 8 mains AND Justin's own Claude access simultaneously (mitigated by the cheap pivot + the intact workspace: interactive access to identities survives anything); **the 1-year `setup-token` on Railway is a concentrated high-value credential** (compromise = fleet impersonation + subscription burn; rotation calendared, scoped to inference only); two harness codebases (contained: the seam is MCP + DB + structured briefs, not shared code); we own a task-tracker UI surface (dashboard tab + Slack ballots) that Linear used to provide.
