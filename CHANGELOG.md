# Changelog — _evryn-meta (cross-repo docs · dashboard · protocols)

**How to use this file:** reasonably concise entries per substantive change to **this repo**. Most recent at top; detail lives in git log.

**What rises to a changelog entry — and who writes it.** An entry records a **deploy or a substantive change** (code, identity, schema, a protocol, a doc, the dashboard) — the unit is a *ship/change*, not a session. **Session-management narration does NOT belong here:** a conductor recounting coordination with its lanes ("answered AC2's notes," re-spin logistics, routine pushes), or in-flight status that hasn't shipped, belongs in handoffs and git — not the changelog. **The AC responsible for a ship owns its entry** and — being contextualized on the work — routes it to each repo the change actually touched.

**Scope — `_evryn-meta`, and read by EVERY AC on load.** This changelog records changes to `_evryn-meta` itself: the Hub + spokes, protocols, ADRs, load cascades, `CLAUDE.md`, and the monitoring dashboard (`_evryn-meta/dashboard`). Every AC skims it on load — it carries changes to their own operating manual, the protocols, and the cascades. **Routing rule:** an entry belongs here **iff the change touched a file in `_evryn-meta`.** A cross-repo change that does **not** touch `_evryn-meta` gets **no** entry here — logging "two other repos changed" is pure noise and doubles what ACs already load per-repo. If a change touched `_evryn-meta` **and** another repo, log the `_evryn-meta` slice here and the other repo's slice there.

**Monthly archive:** at month-end, move the prior month's entries to `docs/historical/changelog-historical/changelog-YYYY-MM.md` (trailing by one month).

**Skim on full-cascade load:** this file's top ~100 lines + section headers (the changelog-skim step in your load cascade points here).

- **Step 78 Half B — BUILT + QC-GO (zero blockers), NOT merged, NOT deployed.** The dashboard's "Evryn Product" tab now renders **5 per-subsystem status lights** (db · slack_socket · poll · m1 · errors_24h) off Half A's new tri-state `/health` — the operator payoff of Half A (see *which* subsystem is unhappy, not just green/red). - **The load-bearing change in the API:** `fetchProcessUp()` → `fetchHealth()`, which now **parses the response body even on a non-200** — a `wedged` arrives as a **503 whose body carries the very checks the lights exist to show**, so the old `if (!res.ok) return null` would have blanked all four lights in precisely the state they were built for. Timeout / `clearTimeout`-in-`finally` / never-throws→`null` all preserved; a new `parseHealthChecks()` safely coerces an untrusted body (absent/malformed → `null`); `processUp` stays backward-compatible so the existing liveness banner is untouched. — [AC3 · DC]
- **Two behaviors verified in BOTH directions (QC, via her own independently-built harness — 10 paths):** **(1) Graceful degrade** — Half A isn't deployed, so live `/health` still returns the OLD shape → the 4 health lights render idle "—", never a false red, never a broken layout. **(2) Halt-subsumes-staleness** — a breaker trip calls `stopPolling()`, so `poll.ok` goes false ~15min later; the poll light renders **amber "expected — M1 halted"** instead of a second false red (one halt reads as one fault), **while a genuine wedge — poll stale, breaker NOT tripped — still renders RED** (hard-asserted regression guard; that alarm must never be softened). This mirrors `decideHealthStatus`'s halted-beats-wedged precedence at the presentation layer. DC surfaced the two-reds problem himself and built to spec rather than deviating silently; AC ruled the fix. — [DC flagged · AC ruled · QC verified]
- **Colour scheme settled + a design-doc self-contradiction closed:** **RED = actual fault** (incl. `m1.breaker_tripped` → RED "HALTED" + reason) · **AMBER = an expected downstream consequence** of another light's fault (only the halt-caused poll case) · **GREEN = healthy** · **GREY = no data**. The Step-78 design doc contradicted itself (its rationale block said the dashboard shows a halt RED; its Half-B section said amber); QC caught it, the build had followed RED, and AC corrected the doc — a halt needs a manual restart, so red is honest and amber would under-call it. — [QC caught · AC ruled]
