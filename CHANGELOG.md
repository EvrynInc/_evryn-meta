## 2026-07-15 (ACf — team-runtime MEMORY + WAKE-ECONOMICS architecture designed, ratified [ADR-052], Phase 1 started; ARCH/BUILD reconciled)

- **The team-runtime memory architecture is designed, ratified, and building.** A Fable deep-dive (full read of the live runtime + the memory corpus + an external agentic-memory SOTA sweep) produced a recommended architecture; Justin reviewed and ruled the same day; **ADR-052** records the decision, and the design-of-record is `evryn-team-workspace/shared/projects/ops/team-runtime/2026.07.15-acf-team-runtime-memory-architecture.md`. **The reframe:** the memory *design* (`consolidation-protocol.md`) was already mature — the gap was that the autonomous runtime never *fires* consolidation (notes pile up) and had no answer to the "meta-meeting token bomb" (8 agents in a thread = 7 full wakes per message). — [ACf · Justin ruled]
- **The load-bearing decisions (Justin's rulings):** (1) **the thread IS the session** — load it in full, drop the 30-msg window; (2) **wake economics = membership, not a model gate** — only agents *in* a thread wake on it; invite/`leave_thread`/late-invite-one-pass, run like a good meeting (this replaced the deep-dive's first-draft Haiku "should I engage?" gate — Justin: deterministic beats a dumb model guessing); (3) **one shared 30-min-refreshed digest per thread + a per-agent scratchpad *headline*** as the personal lens (killed the first draft's per-message tagging/importance/staleness machinery — "convoluted and prone to error"); (4) **private per-thread scratchpad** (relief valve so working-scratch stops bloating `MEMORY.md`); (5) **dormancy memory-write, thread-ID-stamped** (the "echo rule" — a re-activated thread's memory + digest must reconcile as one story); (6) **per-agent wake serialization** ("one mind, one moment" — cross-thread learning merges through memory, never forks); (7) **two-lane consolidation** wiring for Phase 2 (autonomous Lane A commits reversible work; Lane B parks Protected-list items for Justin's async approval — with a HARD requirement that his review be phone-light so he runs it often); (8) **retrieval:** Postgres FTS first, pgvector only on observed failure. — [ACf + Justin]
- **The thread/identity model — "one mind, many rooms" (design doc §4.6):** a thread is a room, a wake is the agent stepping into one room fully briefed; there are no floating "versions" because memory (the trunk) + digests/headlines + the ledger ride into every wake. Cross-room learning is a thread-stamped memory note, not a message-to-self. Threads are bounded by TOPIC (email-subject-line discipline); `post_to_thread` moves context between rooms. Digests (rooms you're in) ≠ the little birds (watchers of rooms you're NOT in). — [ACf, Justin's questions settled]
- **SOTA-vetted (design doc §9):** our narrative-memory-file + human-reviewed-consolidation substrate is mainstream-correct and *ahead* on auditability; the field confirms you must gate ambient wakes (LangChain "ambient-agent triage," 40–70% cost cuts) — we do it via membership rather than a classifier. Dropped at our scale: temporal knowledge graphs (a product-scale tool, flagged for Evryn v0.3+) and speculative vectors. Vendor memory-benchmark deltas = theatre. — [ACf, external sweep]
